Base dati

E' utile avere un'idea approssimativa dell'organizzazione delle informazioni e dei dati nei database. Le unita' di lavoro in TopoDroid sono i rilievi e le calibrazioni. Un rilievo e' una singola sessione di rilievo di un pezzo di grotta. Una calibrazione e' una singola calibrazione DistoX.
I rilievi sono archiviati nel database "distox14.sqlite". C'e' un tale database in ogni cartella di lavoro.
Le calibrazioni sono memorizzate nel database "device10.sqlite", insieme alle informazioni DistoX. Questo database e' globale ed e' presente solo nella cartella "TopoDroid".

  Di conseguenza, la tabella piu' importante nel database del rilievo e':

  • surveys: informazioni sui rilievi

Le registrazioni dei rilievi contengono il nome del rilievo, la data, la squadra, la declinazione magnetica, il nome della stazione iniziale ed eventualmente un commento.
Relativi ai rilievi sono le tabelle:
  • shots : dati del rilievo
  • stations: capisaldi salvati
  • plots : disegni 2D; piante, sezioni, sezioni trasversali
  • photos : immagini associate ai rilievi
  • audios : commenti audio associati a sondaggi
  • sensors : misure associate agli scatti del sondaggio
  • fixeds : (capisaldi geolocalizzati)
  • sketches: modelli 3D

Le tabelle piu' importanti nel database del dispositivo sono:

  • devices: informazioni sui dispositivi
  • calibs : informazioni sulle calibrazioni

  La tabella "calibs" contiene il nome della calibrazione, la data, il dispositivo DistoX, l'algoritmo e i risultati dell'ultimo calcolo: coefficienti, errore medio, errore massimo, numero di iterazioni.

Relativa alle calibrazioni e' la tabella di dati

  • gms: dati di calibrazione

Per concludere, ogni database ha la tabella

  • configurazioni : valori di configurazione

Puoi saperne di piu' sull'organizzazione dei dati nel database esaminandolo con un visualizzatore SQLite.
