Sottodirectory TopoDroid

Ecco alcune informazioni su come vengono memorizzati i dati del rilievo, i dati di calibrazione ed i disegni. Puoi tranquillamente saltare le seguenti e le prossime due sezioni e tornarvi in seguito, se necessario.

L'app crea la sua directory di base predefinita, chiamata "TopoDroid", nella memoria esterna principale (in genere /sdcard). Questa directory non viene rimossa quando si disinstalla l'app. In questa directory l'app crea due database SQLite. Il primo è "device10.sqlite", in cui sono memorizzati i dati di calibrazione, le informazioni DistoX e altri dati. Il secondo è "distox14.sqlite" e memorizza i dati dei rilievi. L'app crea anche sottodirectory per i diversi tipi di file. Queste sottodirectory vengono create solo nella directory di base "TopoDroid":

  • bin: file del firmware DistoX2
  • ccsv: file di testo CSV delle calibrazioni esportate
  • dump: file di memoria DistoX
  • Symbol: strumenti di disegno (punti, linee, aree)
La sottodirectory "symbol" contiene i file degli strumenti di disegno. "bin" è usato per memorizzare i file del firmware, sia i file di fabbrica inclusi in TopoDroid che i dump del firmware letti da un DistoX.

Le seguenti sottodirectory vengono create in ogni directory di lavoro:

  • audio: audio commenti
  • cave: Polygon file
  • cav: Topo file
  • csv: file di testo CSV dei rilievi esportati
  • csx: cSurvey file esportati per cSurvey
  • dat: Compass file esportati per Compass
  • dxf: file DXF esportati per AutoCAD
  • grt: Grottolf file esportati per Grottolf
  • gtx: GHTopo file esportati per GHTopo
  • import: file di rilievi da importare (formati: Therion, Compass, VisualTopo, and PocketTopo)
  • kml: GoogleEarth file esportati per GoogleEarth [A]
  • note: file di annotazioni dei rilievi
  • photo: foto dei rilievi (ogni rilievo ha la sua sottodirectory)
  • plt: OziExplorer file di traccia esportati [A]
  • png: file di disegni esportati
  • srv: Pareti file esportati per rilievi
  • svg: file SVG esportati da disegni
  • sur: WinKarst file esportati per WinKarst
  • svx: Survex file esportati per Survex
  • tdr: file di disegni formato binario
  • th: Therion file esportati per Therion
  • th2: scrap file di Therion
  • th3: file per modelli 3D
  • tmp: per file temporanei. I file in questa directory possono essere cancellati.
  • top: PocketTopo file esportati per PocketTopo
  • tro: VisualTopo file esportati per VisualTopo
  • zip: archivi ZIP (file esportati ed importabili)
Some of these subdirectories are used only to store exported files (cav, cave, csx, dat, dxf, png, svx, th, th2, top, tro). Others are used to store data (tdr, th3, note, photo). The subdirectory "zip" is used to store zipped archives as well to read them and uncompress in TopoDroid. When you want to import a survey made with another program (Therion, Compass, VisualTopo, or PocketTopo) you must place it in the subdirectory "import".

Alcune di queste sottodirectory vengono utilizzate solo per memorizzare i file esportati (cav, cave, csx, dat, dxf, png, svx, th, th2, top, tro). Altre sono usate per memorizzare dati (tdr, th3, note, foto). La sottodirectory "zip" viene utilizzata per salvare i file dei rilievi compressi e per leggerli e decomprimerli in TopoDroid. Quando vuoi importare un rilievo realizzato con un altro programma (Therion, Compass, VisualTopo o PocketTopo) devi posizionarlo nella sottodirectory "import".
